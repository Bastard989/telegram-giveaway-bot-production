# Руководство по использованию Telegram-бота для розыгрышей

Эта инструкция написана для человека без технического опыта. Делайте шаги по порядку.

## 1. Что это за бот

Бот помогает проводить розыгрыши в Telegram-каналах:

- публикует пост розыгрыша в выбранный канал;
- собирает участников по кнопке или по комментариям;
- может проверять подписку на каналы;
- выбирает победителей по призовым местам;
- позволяет вручную назначить победителя по username;
- хранит участников и розыгрыши в базе внутри папки проекта.

Отдельную базу данных устанавливать не нужно. База создаётся как файл внутри папки проекта.

## 2. Что понадобится перед запуском

Нужно заранее подготовить:

1. Токен Telegram-бота из BotFather.
2. Ваш Telegram username или Telegram ID.
3. Канал, где будет опубликован розыгрыш.
4. Каналы, которые будут условиями подписки, если они нужны.

Токен выглядит как длинная строка от BotFather. Никому не отправляйте токен публично.

## 3. Как открыть панель запуска

Откройте папку проекта.

На macOS сначала попробуйте открыть:

```text
1_Запустить_бота_MAC.app
```

Если macOS показывает предупреждение про неизвестного разработчика:

1. Нажмите правой кнопкой мыши по `1_Запустить_бота_MAC.app`.
2. Выберите `Открыть`.
3. В появившемся окне снова нажмите `Открыть`.

Это стандартное окно безопасности macOS для приложений, скачанных не из App Store.

Если `1_Запустить_бота_MAC.app` не открылся, используйте запасной запуск:

```text
START.command
```

Если macOS пишет, что файл `START.command` повреждён или его нельзя открыть, это не значит, что бот сломан. Так macOS блокирует файлы, скачанные из Telegram или браузера.

Что сделать:

1. Откройте папку с ботом в Finder.
2. В верхнем меню Finder нажмите `Сервис`.
3. Нажмите `Новый терминал по адресу папки`.
4. В открывшееся окно Terminal вставьте команду:

```bash
xattr -dr com.apple.quarantine . 2>/dev/null; chmod +x START.command; ./START.command
```

5. Нажмите Enter.

Если пункта `Новый терминал по адресу папки` нет:

1. Откройте приложение `Terminal`.
2. Напишите `cd ` с пробелом в конце.
3. Перетащите папку с ботом в окно Terminal.
4. Нажмите Enter.
5. Вставьте команду выше и снова нажмите Enter.

На Windows дважды нажмите:

```text
START.bat
```

После этого откроется браузерная панель запуска.

## 4. Первичная настройка

В панели заполните основные поля:

1. `Токен бота` - вставьте токен из BotFather.
2. `Telegram ID` или `Username` - укажите, кто будет управлять ботом.
3. `Часовой пояс` - обычно оставьте `Europe/Moscow`.
4. `Фраза для участия через комментарии` - например `Участвую`.
5. `Текст стартового меню` - можно оставить как есть.

Telegram ID указывать не обязательно, если указан username. Но Telegram ID надежнее, потому что username можно поменять.

После заполнения нажмите:

```text
Сохранить настройки
```

Панель запомнит настройки в папке проекта. При следующем открытии токен заново вводить не нужно.

Если сверху написано:

```text
Токен бота: сохранён
```

значит токен уже записан.

## 5. Первый запуск

После сохранения настроек нажмите кнопки по порядку:

1. `Установить нужные файлы`
2. `Подготовить базу данных`
3. `Запустить с сохранёнными настройками`

Кнопку установки нужных файлов обычно достаточно нажать один раз после распаковки проекта.

Кнопка подготовки базы создаёт файл базы внутри папки проекта. Ничего отдельного устанавливать не нужно.

## 6. Повторный запуск в другой день

Если настройки уже сохранены, каждый раз делать всё заново не нужно.

Обычный повторный запуск:

1. На macOS откройте `1_Запустить_бота_MAC.app`, на Windows откройте `START.bat`.
2. Дождитесь открытия панели.
3. Нажмите `Запустить с сохранёнными настройками`.
4. Напишите боту `/start` в Telegram.

Токен и владелец останутся сохранёнными.

## 7. Важно про работу бота

Бот работает только пока запущен процесс на компьютере или сервере.

Если бот лежит на вашем компьютере:

- компьютер должен быть включён;
- интернет должен работать;
- панель/бот не должны быть остановлены.

Если выключить компьютер, бот перестанет отвечать. Для постоянной работы лучше держать его на сервере или VPS.

## 8. Права бота в Telegram-каналах

Есть несколько разных случаев.

### Канал публикации

Это канал, куда бот отправляет пост розыгрыша.

Здесь бот обязательно должен быть администратором, иначе Telegram не даст ему публиковать посты.

Что дать боту:

- право публиковать сообщения;
- право редактировать сообщения желательно, но не обязательно;
- право удалять сообщения не обязательно.

### Строгая проверка подписки

Это условие вида: человек должен подписаться на канал, и бот реально проверяет подписку.

Для строгой проверки бот должен быть администратором в канале-условии. Иначе Telegram не даст проверить, подписан человек или нет.

Используйте этот вариант, если владелец канала-условия готов добавить бота в администраторы.

### Мягкая ссылка без проверки

Это условие вида: бот показывает кнопку `Подписаться`, но не проверяет подписку.

Для мягкой ссылки бот не обязан быть администратором в канале-условии.

Минус: человек технически может нажать `Участвовать`, даже если не подписался.

Используйте этот вариант, если канал чужой или вы не хотите добавлять туда бота администратором.

### Розыгрыш по комментариям

Если розыгрыш собирает участников через комментарии, у канала должна быть подключена группа обсуждения.

Бот должен быть администратором:

- в канале публикации;
- в группе обсуждения.

Иначе он не сможет нормально читать комментарии и добавлять участников.

## 9. Создание розыгрыша

Откройте Telegram и напишите своему боту:

```text
/start
```

Нажмите:

```text
Создать розыгрыш
```

Дальше бот спросит данные по шагам.

### Шаг 1. Тип розыгрыша

Выберите один вариант:

- `По кнопке` - участники нажимают кнопку участия под постом.
- `По комментариям` - участники пишут фразу в комментариях.

Для первого теста проще выбрать `По кнопке`.

### Шаг 2. Название

Введите название розыгрыша. Это название будет видно вам в списке розыгрышей.

Пример:

```text
Розыгрыш сертификата
```

### Шаг 3. Текст поста

Введите текст, который будет опубликован в канале.

Пример:

```text
Дарим сертификат на 5000 рублей.
Чтобы участвовать, нажмите кнопку ниже.
```

### Шаг 4. Фото или видео

Отправьте фото/видео для поста.

Если медиа не нужно, напишите:

```text
нет
```

### Шаг 5. Дата окончания

Введите дату в формате:

```text
ДД.ММ.ГГГГ ЧЧ:ММ
```

Пример:

```text
30.06.2026 20:00
```

Дата должна быть в будущем.

### Шаг 6. Количество призовых мест

Это сколько мест будет в результатах.

Примеры:

- `1` - только первое место;
- `3` - первое, второе и третье место;
- `10` - места с 1 по 10.

### Шаг 7. Победителей на каждое место

Это сколько людей попадёт на одно место.

Примеры:

- 5 призовых мест и 1 победитель на место = 5 победителей.
- 5 призовых мест и 2 победителя на место = 10 победителей.
- 10 призовых мест и 3 победителя на место = 30 победителей.

### Шаг 8. Капча

Капча нужна, чтобы уменьшить количество случайных или автоматических участий.

Для простого розыгрыша можно выбрать `Нет`.

## 10. Настройка канала публикации

После создания розыгрыша откройте карточку розыгрыша и нажмите:

```text
Канал публикации
```

Отправьте один из вариантов:

- username канала, например `@my_channel`;
- ссылку на канал, например `https://t.me/my_channel`;
- ссылку на пост, например `https://t.me/my_channel/123`;
- пересланный пост из канала.

Бот должен быть администратором этого канала.

## 11. Добавление условий подписки

В карточке розыгрыша нажмите:

```text
Добавить условие подписки
```

Выберите тип условия:

- `Строгая проверка подписки`;
- `Мягкая ссылка без проверки`.

Потом отправьте канал:

- `@username`;
- ссылку `t.me/...`;
- ссылку на пост;
- пересланный пост из канала.

Можно добавить несколько условий.

## 12. Запуск розыгрыша

Когда канал публикации и условия добавлены, нажмите:

```text
Запустить
```

Бот опубликует пост в канале.

Если Telegram выдаёт ошибку, чаще всего причина одна из трёх:

- бот не админ в канале публикации;
- не добавлено ни одного условия;
- канал указан неправильно.

## 13. Участие пользователей

Для розыгрыша по кнопке пользователь нажимает кнопку участия под постом.

Для розыгрыша по комментариям пользователь пишет фразу участия в комментариях. Фраза задаётся в панели, по умолчанию:

```text
Участвую
```

Если включена строгая проверка подписки, бот проверит подписку перед добавлением участника.

## 14. Статистика и экспорт участников

В активном розыгрыше доступны кнопки:

- `Статистика` - посмотреть количество участников;
- `Экспорт участников CSV` - скачать таблицу участников;
- `Завершить сейчас` - завершить розыгрыш вручную;
- `Остановить` - остановить розыгрыш без выбора новых участников.

CSV-файл можно открыть в Excel, Numbers или Google Sheets.

## 15. Ручной выбор победителя

Для активного розыгрыша нажмите:

```text
Выбрать победителя вручную
```

Отправьте username участника:

```text
@username
```

Важно:

- пользователь уже должен быть среди участников розыгрыша;
- ручной победитель становится одним из победителей первого места;
- остальные места бот заполнит автоматически из других участников;
- если у пользователя нет username, выбрать его вручную по username нельзя.

## 16. Завершение розыгрыша

Розыгрыш завершится автоматически по дате окончания.

Также можно завершить вручную:

```text
Завершить сейчас
```

После завершения бот:

- выберет победителей;
- отправит результаты владельцу;
- отправит результаты в канал публикации, если сможет.

Если участников меньше, чем победных мест, бот выберет столько победителей, сколько есть участников.

## 17. Что нельзя делать

Не делайте это без необходимости:

- не удаляйте файл `.env`, там сохранены настройки;
- не удаляйте папку `runtime`, там база и рабочие файлы;
- не запускайте несколько копий одного и того же бота с одним токеном;
- не публикуйте токен бота в чатах, скриншотах и документах;
- не меняйте файлы кода вручную, если не понимаете зачем;
- не закрывайте компьютер, если бот должен работать прямо сейчас.

## 18. Что можно удалять безопасно

Обычно можно удалять:

- файлы `.log` в папке `runtime`;
- файлы `.pid` в папке `runtime`, если бот точно остановлен;
- папку `exports`, если старые CSV-выгрузки не нужны.

Не удаляйте базу `runtime/giveaway-bot.sqlite3`, если хотите сохранить старые розыгрыши и участников.

## 19. Передача бота другому человеку

Если передаёте чистую копию заказчику, отправляйте zip:

```text
Бот_для_розыгрышей_Telegram.zip
```

В чистом zip нет вашего токена и вашей базы.

Заказчик должен:

1. Распаковать zip.
2. Открыть файл `РУКОВОДСТВО_ПО_ИСПОЛЬЗОВАНИЮ.md`.
3. На macOS открыть `1_Запустить_бота_MAC.app`, на Windows открыть `START.bat`.
4. Вставить свой токен.
5. Указать свой username или Telegram ID.
6. Нажать `Сохранить настройки`.
7. Нажать `Установить нужные файлы`.
8. Нажать `Подготовить базу данных`.
9. Нажать `Запустить с сохранёнными настройками`.

Если нужно передать уже настроенного бота вместе с историей розыгрышей, отдельно передавайте `.env` и `runtime/giveaway-bot.sqlite3`. Делайте это только доверенному человеку.

## 20. Быстрый чек-лист перед первым розыгрышем

Проверьте:

- бот запущен в панели;
- в Telegram бот отвечает на `/start`;
- бот добавлен админом в канал публикации;
- выбран канал публикации;
- добавлено хотя бы одно условие;
- для строгой проверки бот админ в канале-условии;
- дата окончания указана в будущем;
- количество мест и победителей введено числами.

## 21. Возможные ошибки и решения

### Mac не открывает 1_Запустить_бота_MAC.app

Причина:

- приложение скачано из Telegram или браузера;
- приложение не подписано через Apple Developer;
- macOS показывает стандартную защиту от неизвестного разработчика.

Решение:

1. Нажмите правой кнопкой мыши по `1_Запустить_бота_MAC.app`.
2. Выберите `Открыть`.
3. В появившемся окне снова нажмите `Открыть`.

Если macOS всё равно пишет, что файл повреждён, используйте запасной вариант ниже.

### Mac пишет, что START.command повреждён

Причина:

- macOS заблокировала файл после скачивания из Telegram или браузера.

Решение:

1. Откройте папку с ботом в Finder.
2. В верхнем меню Finder нажмите `Сервис`.
3. Нажмите `Новый терминал по адресу папки`.
4. Вставьте команду:

```bash
xattr -dr com.apple.quarantine . 2>/dev/null; chmod +x START.command; ./START.command
```

5. Нажмите Enter.

Эта команда снимает блокировку macOS, разрешает запуск файла и открывает панель бота. Токены и настройки она не удаляет.

### Панель не открывается

Причина:

- файл `START.command` или `START.bat` не был запущен;
- macOS заблокировала запуск файла;
- Python не установлен.

Решение:

- если macOS пишет, что `START.command` повреждён, выполните инструкцию из раздела выше;
- на macOS можно открыть терминал в папке проекта и выполнить `./START.command`;
- если команда не запускается, выполните `python3 control_panel.py`;
- установите Python 3.10 или новее.

### Страница `127.0.0.1:8088` не открывается

Причина:

- панель запуска не работает;
- панель выбрала другой свободный порт.

Решение:

- запускайте `START.command` или `START.bat`;
- не вводите порт вручную;
- смотрите адрес, который открылся автоматически.

### Кнопка подготовки базы данных не работает

Причина:

- нет прав на запись в папку проекта;
- папка проекта лежит в защищённом месте;
- файл базы открыт или заблокирован системой.

Решение:

- перенесите проект в обычную папку, например Desktop или Documents;
- закройте программы, которые могли открыть файл базы;
- нажмите `Подготовить базу данных` снова.

### Бот не запускается

Причина:

- не указан токен бота;
- не указан владелец бота;
- база данных не подготовлена;
- зависимости не установлены.

Решение:

1. Укажите токен из BotFather.
2. Укажите Telegram ID или username владельца.
3. Нажмите `Установить нужные файлы`.
4. Нажмите `Подготовить базу данных`.
5. Нажмите `Запустить с сохранёнными настройками`.

### Бот не отвечает в Telegram

Причина:

- процесс бота не запущен;
- токен неправильный;
- компьютер выключен или без интернета;
- бот запущен на другом компьютере с тем же токеном.

Решение:

- проверьте статус в панели запуска;
- перезапустите бота;
- проверьте токен в BotFather;
- не запускайте несколько копий одного и того же бота одновременно.

### Бот не публикует розыгрыш в канал

Причина:

- бот не админ в канале публикации;
- канал публикации не выбран;
- канал указан неправильно.

Решение:

- добавьте бота админом в канал публикации;
- в карточке розыгрыша нажмите `Канал публикации`;
- отправьте username канала или ссылку на канал.

### Бот не проверяет подписку

Причина:

- условие добавлено как строгая проверка, но бот не админ в канале условия;
- пользователь реально не подписан;
- канал указан неправильно.

Решение:

- для строгой проверки добавьте бота админом в канал условия;
- если админку давать нельзя, используйте мягкую ссылку;
- попросите участника подписаться и нажать участие снова.

### Ручной победитель не находится

Причина:

- пользователь не участвовал в розыгрыше;
- username введён с ошибкой;
- у пользователя нет username.

Решение:

- попросите пользователя сначала нажать кнопку участия;
- проверьте username;
- если у пользователя нет username, ручной выбор по username невозможен.

### Победителей меньше, чем ожидалось

Причина:

- участников меньше, чем рассчитанное количество победителей.

Пример:

- задано 5 мест и 2 победителя на место;
- нужно 10 победителей;
- участвовало только 7 человек;
- бот выберет только 7 победителей.

Решение:

- уменьшите количество мест или победителей на место;
- дождитесь большего количества участников.
