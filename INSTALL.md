# Установка и запуск

Эта инструкция для человека, который скачал готовый zip с Telegram-ботом для розыгрышей.

## Что нужно заранее

1. Установленный Python 3.
2. Токен Telegram-бота из BotFather.
3. Ваш Telegram ID или username.
4. Telegram-канал, где бот будет публиковать розыгрыши.

Отдельную базу данных устанавливать не нужно. База создаётся автоматически как файл внутри папки проекта.

## macOS

1. Распакуйте zip.
2. Откройте распакованную папку.
3. Убедитесь, что в ней есть файл `START.command`.
4. Откройте Terminal именно в этой папке.

Можно открыть Terminal, написать `cd ` с пробелом, перетащить папку проекта в окно Terminal и нажать Enter.

5. Выполните команду:

```bash
xattr -dr com.apple.quarantine . 2>/dev/null; chmod +x START.command; ./START.command
```

Команда снимает блокировку macOS с файлов, разрешает запуск `START.command` и открывает локальную панель бота.

Если команда пишет, что `START.command` не найден, значит Terminal открыт не в папке проекта.

## Windows

1. Распакуйте zip.
2. Откройте папку проекта.
3. Запустите:

```text
START.bat
```

Если Windows показывает предупреждение безопасности, нажмите `Подробнее`, затем `Выполнить в любом случае`.

## Настройка в панели

После запуска откроется браузерная панель.

1. Вставьте `Токен бота`.
2. Укажите `Telegram ID` или `Username` владельца.
3. Нажмите `Сохранить настройки`.
4. Нажмите `Установить нужные файлы`.
5. Нажмите `Подготовить базу данных`.
6. Нажмите `Проверить окружение`.
7. Нажмите `Запустить с сохранёнными настройками`.
8. Откройте Telegram и отправьте своему боту `/start`.

## Повторный запуск

После первой настройки токен сохраняется локально в `.env`.

В следующий раз:

1. Запустите `START.command` на macOS или `START.bat` на Windows.
2. Нажмите `Запустить с сохранёнными настройками`.

## Backup базы

Перед важными изменениями нажмите:

```text
Backup базы
```

Копии базы сохраняются в:

```text
runtime/backups
```

## Перед передачей проекта другому человеку

Если вы передаёте рабочую папку, сначала нажмите:

```text
Очистить токен перед передачей
```

Лучше передавать не рабочую папку, а чистый release zip. В чистый zip не попадают `.env`, база, логи и служебные файлы.

## Частые ошибки

### macOS пишет, что файл нельзя открыть

Запустите команду из папки проекта:

```bash
xattr -dr com.apple.quarantine . 2>/dev/null; chmod +x START.command; ./START.command
```

### Панель пишет, что токен невалидный

Проверьте токен в BotFather. Если токен кому-то отправляли, перевыпустите его.

### Бот не публикует в канал

Добавьте бота администратором в канал публикации и дайте право публиковать сообщения.

### Строгая проверка подписки не работает

Для строгой проверки бот должен быть администратором в канале-условии.

### Бот перестал отвечать

Проверьте, что компьютер или сервер включён, интернет работает, а бот запущен в панели.
