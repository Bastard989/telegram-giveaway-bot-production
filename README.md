# Telegram Giveaway Bot

Telegram Giveaway Bot - автономный бот для проведения розыгрышей в Telegram-каналах. Он помогает создать розыгрыш, опубликовать пост, собрать участников, проверить условия подписки, выбрать победителей и выгрузить результаты без отдельного сервера базы данных.

Проект рассчитан на практичный запуск: локальная браузерная панель управляет настройками, зависимостями, базой данных и процессом бота. Данные хранятся в SQLite внутри папки проекта, поэтому систему можно запускать локально, на VPS или на любом сервере с Python.

## Что умеет

- Создавать розыгрыши через Telegram-меню администратора.
- Публиковать посты розыгрышей в выбранный Telegram-канал.
- Добавлять к посту фото, видео или GIF и сохранять Telegram-форматирование, включая цитаты.
- Собирать участников по кнопке под постом.
- Собирать участников по комментариям с заданной ключевой фразой.
- Проверять подписку на каналы, где бот добавлен администратором.
- Добавлять мягкие условия подписки по ссылке без технической проверки.
- Настраивать несколько призовых мест и количество победителей на каждое место.
- Выбирать победителей автоматически из списка участников.
- Предварительно назначать победителя первого места по username без остановки конкурса.
- Экспортировать участников в CSV.
- Делать backup локальной базы данных.
- Запускаться и останавливаться через локальную web-панель.
- Собирать чистый release zip без токенов, логов, кешей и локальной базы.

## Для чего подходит

Бот полезен для Telegram-каналов, сообществ, медиа, образовательных проектов, брендов и администраторов, которым нужен контролируемый процесс розыгрышей:

- розыгрыши среди подписчиков канала;
- промо-акции с несколькими призовыми местами;
- конкурсы с участием через комментарии;
- партнерские розыгрыши с условиями подписки;
- быстрый экспорт участников для проверки или отчетности.

Для постоянной работы лучше запускать бота на VPS или другом сервере, который не выключается. На домашнем компьютере бот работает только пока компьютер включен, есть интернет и процесс бота запущен.

## Как это устроено

```text
Telegram admin menu -> Giveaway Bot -> Telegram channel post
                                   -> SQLite database
                                   -> CSV export / backups

Local browser panel -> settings, dependencies, database, start/stop
```

Основные части проекта:

- `app.py` - запуск Telegram-бота;
- `control_panel.py` - локальная браузерная панель управления;
- `handlers/` - Telegram-сценарии и бизнес-логика;
- `database/` - модели и настройки базы;
- `runtime/` - локальная база, логи, pid-файлы и backup-файлы;
- `scripts/build_release_zip.py` - сборка чистого zip-архива;
- `tests/` - unit-тесты ключевой логики и панели.

## Требования

- Python 3.10 или 3.11 (рекомендуется 3.11).
- Telegram bot token из BotFather.
- Telegram ID или username владельца бота.
- Telegram-канал, где бот будет публиковать розыгрыши.
- Права администратора для бота в канале публикации.

Отдельный PostgreSQL, MySQL или другой сервер базы данных не нужен. По умолчанию используется SQLite-файл:

```text
runtime/giveaway-bot.sqlite3
```

## Быстрый старт

Скачайте архив проекта, распакуйте его и откройте папку с файлами запуска.

### macOS

Выполните команду из папки проекта:

```bash
xattr -dr com.apple.quarantine . 2>/dev/null; chmod +x START.command; ./START.command
```

Если Terminal открыт не в папке проекта, сначала перейдите в нее:

```bash
cd /path/to/project
```

Затем повторите команду запуска.

### Windows

Запустите файл:

```text
RUN_BOT_WINDOWS.bat
```

Если Windows покажет предупреждение безопасности, нажмите `Подробнее`, затем `Выполнить в любом случае`.

`RUN_BOT_WINDOWS.bat` оставляет диагностическое окно открытым. Если панель не запустится, ошибка останется в этом окне. Если Python не найден, стартовый скрипт предложит установить Python 3.11 через Windows Package Manager или открыть страницу загрузки Python.

## Первый запуск

После запуска откроется локальная браузерная панель.

1. Вставьте токен бота из BotFather.
2. Укажите Telegram ID владельца или username владельца.
3. При необходимости измените часовой пояс, стартовый текст и фразу для участия через комментарии.
4. Нажмите `Сохранить настройки`.
5. Нажмите `Установить нужные файлы`.
6. Нажмите `Подготовить базу данных`.
7. Нажмите `Проверить окружение`.
8. Нажмите `Запустить с сохранёнными настройками`.
9. Откройте Telegram и отправьте боту `/start`.

## Ручной выбор победителя

В активном розыгрыше кнопка `Предварительно выбрать победителя` фиксирует выбранного участника на первом месте, но не завершает конкурс. После выбора бот продолжает принимать новых участников. При наступлении даты окончания или нажатии `Завершить сейчас` предварительный победитель сохраняется, остальные места бот заполняет автоматически и только затем публикует результаты.

Подробный сценарий для ручной настройки описан в:

```text
РУКОВОДСТВО_ПО_ИСПОЛЬЗОВАНИЮ.md
```

Краткая инструкция по установке:

```text
INSTALL.md
```

## Настройки окружения

Панель сохраняет настройки в локальный `.env`. Этот файл содержит секреты и не должен попадать в публичный репозиторий.

Пример:

```env
BOT_TOKEN=put_botfather_token_here
DATABASE_URL=sqlite://runtime/giveaway-bot.sqlite3
OWNERS=123456789
OWNER_USERNAMES=
TIMEZONE=Europe/Moscow
START_TEXT=Главное меню:
COMMENT_GIVEAWAY_KEYWORD=Участвую
```

Параметры:

- `BOT_TOKEN` - токен Telegram-бота из BotFather.
- `DATABASE_URL` - путь к базе данных.
- `OWNERS` - Telegram ID администраторов через запятую.
- `OWNER_USERNAMES` - username администраторов через запятую.
- `TIMEZONE` - часовой пояс для дат и времени.
- `START_TEXT` - текст стартового меню.
- `COMMENT_GIVEAWAY_KEYWORD` - фраза для участия в розыгрыше через комментарии.

## Права бота в Telegram

### Канал публикации

Бот должен быть администратором канала, куда публикуется розыгрыш. Минимально нужно право публиковать сообщения.

### Строгая проверка подписки

Для технической проверки подписки бот должен быть администратором в канале-условии. Это ограничение Telegram API: без прав администратора бот не сможет надежно проверить, подписан ли пользователь.

### Мягкое условие подписки

Мягкое условие показывает пользователю ссылку на канал, но не проверяет подписку технически. Такой режим подходит, когда невозможно добавить бота администратором в канал-условие.

### Розыгрыш по комментариям

Для участия через комментарии у канала должна быть подключена группа обсуждения. Бот должен быть администратором в канале и в группе обсуждения.

## Локальная панель

Панель запускается через `control_panel.py` и открывается в браузере. Через нее можно:

- сохранить токен и владельцев;
- установить зависимости в `.venv`;
- подготовить SQLite-базу;
- проверить Python, зависимости, Telegram-токен и базу;
- запустить бота;
- остановить бота;
- сделать backup базы;
- очистить сохраненный токен из `.env`.

Backup-файлы сохраняются в:

```text
runtime/backups
```

## Сборка release zip

Чтобы собрать чистый архив проекта:

```bash
python3 scripts/build_release_zip.py
```

Или откройте:

```text
BUILD_ZIP.command
```

Архив создается здесь:

```text
dist/Бот_для_розыгрышей_Telegram.zip
```

В архив не попадают:

- `.env`;
- `.venv/`;
- `.git/`;
- `runtime/`;
- `dist/`;
- `tests/`;
- `scripts/`;
- логи, pid-файлы, локальные базы и Python-кеши.

## Разработка

Установите зависимости:

```bash
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
```

Запустите панель:

```bash
.venv/bin/python control_panel.py
```

Запустите бота напрямую:

```bash
.venv/bin/python app.py
```

Unit-тесты:

```bash
.venv/bin/python -m unittest discover -s tests
```

Проверка компиляции:

```bash
.venv/bin/python -m py_compile app.py control_panel.py config/*.py database/*.py database/models/*.py handlers/*.py middlewares/*.py scripts/*.py tests/*.py
```

## Безопасность

- Не публикуйте `.env`.
- Не отправляйте токен BotFather в чатах, issue, логах и скриншотах.
- Если токен попал в чужие руки, перевыпустите его в BotFather.
- Перед публикацией release zip проверьте, что внутри нет `.env`, локальной базы, логов и runtime-файлов.
- Для публичного репозитория держите секреты только локально или в защищенных переменных окружения.

Дополнительные рекомендации описаны в `SECURITY.md`.
