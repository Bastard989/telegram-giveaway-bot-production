# Security Policy

## Секреты

В репозиторий и release zip не должны попадать:

- `.env`;
- токен BotFather;
- локальная SQLite-база;
- логи;
- pid-файлы;
- `.venv`;
- кеши Python;
- временные экспорты с пользовательскими данными.

Файл `.env.example` содержит только шаблонные значения.

## Перед публикацией

1. Нажмите в панели `Очистить сохранённый токен`, если работали с реальным токеном.
2. Соберите чистый архив:

```bash
python3 scripts/build_release_zip.py
```

3. Проверьте, что в архиве нет `.env`, `runtime/`, `.venv/`, `.git/`, логов и SQLite-файлов.
4. Если токен был отправлен в чат, попал на скриншот или был доступен третьим лицам, перевыпустите токен в BotFather.

## Проверка архива

Можно проверить содержимое zip так:

```bash
python3 -c "import zipfile; z=zipfile.ZipFile('dist/Бот_для_розыгрышей_Telegram.zip'); print('\\n'.join(z.namelist()))"
```

В списке не должно быть:

- `.env`;
- `runtime/giveaway-bot.sqlite3`;
- `runtime/*.log`;
- `runtime/*.pid`;
- `.venv/`;
- `.git/`.

## Исторические секреты

Если настоящий токен уже был закоммичен и отправлен в публичный GitHub, недостаточно удалить файл новым коммитом. Нужно:

1. перевыпустить токен в BotFather;
2. очистить историю Git;
3. force-push только после понимания последствий для всех клонов репозитория.

В текущей поставке токен хранится локально в `.env`, который игнорируется Git и исключается из release zip.
